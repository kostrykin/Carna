Detailed Description

Wraps and represents an OpenGL context.

This is a good explanation of the essence of OpenGL contexts:

‍An OpenGL context represents many things. A context stores all of the state associated with this instance of OpenGL. [...] Each context can represent a separate viewable surface, like a window in an application. Contexts can share many kinds of objects between each other. Any OpenGL object types which are not containers are sharable, as well as [...] GLSL Objects. [...] In order for any OpenGL commands to work, a context must be current; all OpenGL commands affect the state of whichever context is current. The current context is a thread-local variable, so a single process can have several threads, each of which has its own current context.

Reference: https://www.opengl.org/wiki/OpenGL_Context

Carna assumes that all OpenGL contexts within your application are capable of sharing objects. Carna does not support multi-threading environments, i.e. all Carna-related code must be run on the same thread. The states are maintained through the RenderState class.
