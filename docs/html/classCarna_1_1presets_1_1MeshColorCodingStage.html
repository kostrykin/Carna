Detailed Description

Implements a simple interface for mapping frame coordinates to base::Mesh objects.

Lets assume we have an OpaqueRenderingStage defined, e.g.:

opaque = new presets::OpaqueRenderingStage( GEOMETRY_TYPE_OPAQUE );
renderer->appendStage( opaque );

The concept of geometry types is explained here.

Than we can instantiate the MeshColorCodingStage like this:

mccs = new presets::MeshColorCodingStage();
mccs->putGeometryType( GEOMETRY_TYPE_OPAQUE, presets::OpaqueRenderingStage::ROLE_DEFAULT_MESH );
renderer->appendStage( mccs );

After rendering the frame, we can query the MeshColorCodingStage object for meshes at particular frame coordinates using its pick method.

Note
In the rendering process this stage can be inserted anywhere.
See also
The base::SpatialMovement class in combination with MeshColorCodingStage makes the implementation of drag-&-drop like behaviour for base::Spatial objects very easy.

Enables support for materials if geometryType has been activated or will be activated later. Otherwise this method has no effect.

The behaviour that this enables is the following at the moment and may be extended in the future. If the stage encounters a mesh with base::ManagedMeshBase::primitiveType set to base::IndexBufferBase::PRIMITIVE_TYPE_POINTS, it queries its material for a pointSize parameter that it than takes account of.

Called once per pass.

If this rendering stage maintains one or more Carna::base::RenderQueue objects, than this is the right place to build them. Note that the queues need to be rebuilt only once per frame and not per pass, unless isViewTransformFixed is false. If it is true and this is not the first invocation of this method since the last time prepareFrame was called, rewinding the queue will be sufficient.
